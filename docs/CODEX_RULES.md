# Codex 開發規則

這份文件給新的 Codex / ChatGPT 聊天室接手使用。請先讀完再改任何功能。

## 核心不要亂改

除非使用者明確要求，請不要改以下核心：

- 不要改調音器核心。
- 不要改 `detectorV2`。
- 不要改 Fine Pitch Tracker。
- 不要改 cents 偵測。
- 不要改長音練習偵測核心。
- 不要改每日目標 / streak / freeze 既有邏輯。
- 不要改精靈花園水滴系統，除非任務明確要求。
- 不要改圖鑑 localStorage 結構。
- 不要改首頁展示植物邏輯，除非任務明確要求。
- 不要改 sound manager。
- 不要重做音效邏輯。
- 不要清空 localStorage。
- 不要重置使用者水滴、植物、圖鑑、每日目標或連續練習資料。
- 不要把手機 App 做成桌機版。

## 修改方式

- 每次修改都要小範圍處理。
- 先讀現有程式碼與現有 class / helper，再依照既有模式改。
- 不要為單一需求做大重構。
- 不要順手改使用者沒有要求的功能。
- 不要大量重新命名 class、id、localStorage key。
- 不要改動不相關圖片、音效、manifest、service worker。
- 不要為了 UI 小調整改掉資料邏輯。

## 版本更新規則

每次完成實際功能、UI、資源或快取相關修改時，請同步更新：

- App 內顯示版本號。
- CSS/JS query version。
- service worker cache name。

請避免只改其中一個，造成手機 PWA 卡舊版。

如果只是新增純文件，例如 `docs/*.md`，不需要更新版本號或 service worker cache。

## 手機優先規則

- 以手機直式畫面為主要使用場景。
- 不要設計桌機大儀表板。
- 不要讓內容只能靠桌機寬度才舒服。
- 快速入口、設定、練習室、花園、圖鑑都要優先檢查手機尺寸。
- 文字不可爆版，不可互相重疊。
- 底部導航不可遮住重要操作。

## 視覺風格規則

保持：

- 日系手帳感。
- 暖米色背景。
- 木棕色文字與主按鈕。
- 琥珀金點綴。
- 插畫感、柔和陰影、淡紙感。
- 可愛但不幼稚。

避免：

- 科技藍。
- 高飽和霓虹。
- 手遊抽卡感。
- 電商商城感。
- 大量粒子與漂浮裝飾。
- 冷冰冰系統設定頁。

## 圖片與素材規則

- 所有正式圖片、音效、字體資源都應放在專案內。
- 不要依賴 `/mnt/data`。
- 不要依賴 `/tmp`。
- 不要依賴外部臨時圖片連結。
- 圖片若要進 App，請複製到 `public/assets/` 底下合適位置。
- 去背圖要用深色或黑色檢查圖確認。
- 去背時要特別保護：
  - 眼睛。
  - 嘴巴。
  - 手臂。
  - 臉部陰影。
  - 腮紅。
  - 口琴細節。
- 不要把角色五官、手臂或陰影誤刪。
- 使用者常會用紅圈標出要去背處；紅圈是標示，不是素材本身。

## 音效規則

- 不要重做 sound manager。
- 不要新增音效播放架構。
- 如果要新增音效，只接入現有 sound manager。
- 尊重聲音設定：
  - App 音效關閉時，不播放任何音效與花園 BGM。
  - 完成音效、澆水音效、進化音效、花園 BGM 要各自受設定控制。
- 不要破壞既有音效播放時機。

## 花園與圖鑑資料規則

- 不要清空圖鑑。
- 不要改變已採收植物資料。
- 不要重置目前植物。
- 不要重置 featuredSpiritId。
- 圖鑑格位是固定的。
- 初始植物與採收後下一株可以隨機，但解鎖圖鑑時仍回到該 species 的固定位置。
- 所有植物成熟採收後才進圖鑑。
- 尚未種植或尚未解鎖的植物顯示問號圖。

## Git 規則

- 推上 GitHub 前先看 `git status`。
- 不要還原使用者改過但與本任務無關的檔案。
- 不要使用 `git reset --hard`。
- 如果使用者要求「拉遠端整合後再推」，先 pull / rebase 或依現場狀態處理衝突。
- commit 訊息要描述實際修改，不要混入未完成事項。

## 檢查規則

常見檢查項目：

- 手機直式畫面。
- 首頁 hero 與快速入口。
- 全音域調音器。
- 開始練習頁。
- 長音練習室。
- 每日目標。
- 精靈花園。
- 圖鑑 modal。
- 設定頁。
- service worker 是否有版本更新。

如果只是新增文件，不需要跑功能測試，但要確認文件路徑正確。
