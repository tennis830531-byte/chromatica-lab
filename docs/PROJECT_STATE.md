# Chromatic Harmonica Lab 專案狀態

## 基本資訊

- App 名稱：Chromatic Harmonica Lab
- 中文名稱：半音階口琴練習室
- 專案性質：手機優先的 PWA / Web App，未來預計用 Capacitor 包成 Android 與 iOS App。
- 主要檔案：
  - `index.html`
  - `styles.css`
  - `app.js`
  - `manifest.json`
  - `service-worker.js`
  - `public/assets/`

## 設計方向

- 整體風格：日系手帳感、暖米色、木棕色、琥珀金。
- 視覺語氣：可愛但不幼稚，溫暖、柔和、像音樂練習筆記。
- 版面原則：手機優先，不要做成桌機大版面。
- UI 元件方向：
  - 卡片柔和圓角與淡陰影。
  - 標題可帶手帳感，內文保持清楚易讀。
  - 不使用科技藍、大面積冷色、高飽和霓虹、遊戲商城感。
  - 避免過度裝飾，功能入口要清楚。

## 主要功能

- 全音域調音器
  - 顯示最接近音名與 cents 誤差。
  - 有 A4 調音基準說明。
  - 麥克風權限與音準偵測核心已存在，非明確任務不要改。

- 長音練習室
  - 已有長音練習偵測、循環、BPM、完成流程。
  - 每日目標與水滴系統會依完成循環更新。
  - 不要改長音練習偵測核心，除非任務明確要求。

- 音程練習室
  - 目前是下一個重點功能。
  - 定位為「看譜練習 + 調性內音程移動 + 自我判斷 + 節拍引導」。
  - 不接入 detectorV2 / Fine Pitch Tracker / cents 判斷。

- 每日目標
  - 目前以每日練習任務與完成狀態為主。
  - 已調整方向：同類練習需完成不同組合兩次才算該項任務完成。
  - 完成當日所有每日任務可額外獲得 20 滴水滴。

- 連續練習 / streak / freeze
  - 已有連續練習與 freeze 機制。
  - 連續學習 3 天可獲得 1 張 freeze，最多持有 2 張。
  - freeze 可維持 streak，但不應直接觸發里程碑 bonus。

- 精靈花園
  - 使用練習獲得的水滴澆灌植物精靈。
  - 有目前植物、階段進度、今日練習水滴、圖鑑、首頁展示植物。
  - 第一次進入花園時有初始植物選擇流程。
  - 首頁展示植物會左右搖擺與跳躍，點擊時只播放動作，不跳轉花園。

- 植物精靈圖鑑
  - 圖鑑位置固定。
  - 即使新使用者初始選到第 5 隻，採收後也解鎖第 5 格，不改排列。
  - 未被種植 / 尚未解鎖的植物顯示問號圖。

- 設定頁
  - 已整理為多區塊設定。
  - 區塊包含收音、練習、聲音、顯示與動畫、通知與語言、會員帳號、關於。
  - Google 登入目前只是 UI 預留，不做假登入。

- 全球排行榜
  - 首頁入口已預留。
  - 目前只顯示尚未公開，不做排名列表。

## 目前植物精靈與階段名稱

目前前五種植物精靈：

1. 旋律芽芽 → 旋律葉靈 → 旋律森使
2. 菇鳴靈 → 菇鳴樂手 → 菇鳴賢者
3. 花樂精靈 → 花樂舞靈 → 花樂仙子
4. 幸葉芽 → 幸葉靈 → 幸葉祭司
5. 竹芽童 → 竹音童 → 竹音翁

注意：
- 花園頁面目前植物名稱要隨階段改變。
- 圖鑑 modal 點選不同階段時，標題也要跟著改成該階段名稱。
- 若使用者點「編輯」改名，才使用自訂名稱。

## 精靈花園水滴與成長規則

### 水滴來源

- 每日首次進入 App：5 滴。
  - 不計入每日 80 滴練習水滴上限。
  - 不推進 streak。
  - 不算每日任務。

- 完成練習循環：
  - 1 個循環 = 1 滴。
  - 單次練習最高 8 滴。
  - 每日透過練習最多 80 滴。
  - 80 滴只限制「今日透過練習獲得的水滴」，不限制水滴庫存總量。

- 完成當日所有每日任務：
  - 額外 20 滴。
  - 不計入每日 80 滴練習水滴上限。

- 連續練習里程碑：
  - 7 天：50 滴。
  - 30 天：100 滴。
  - 60 天：120 滴。
  - 90 天：150 滴。
  - 120 天起每 30 天：150 滴。
  - 里程碑只能在有效練習後檢查並發放。

- 雨天事件：
  - 進入精靈花園頁面時檢查。
  - 每天最多抽一次。
  - 目前使用者要求機率改為 50%。
  - 不計入每日 80 滴練習水滴上限。
  - 不推進 streak、不完成任務、不觸發里程碑。

### 植物成長

每株植物總共需要 530 滴成熟：

- 第 1 階到第 2 階：100 滴。
- 第 2 階到第 3 階：180 滴。
- 第 3 階到成熟可採收：250 滴。

階段判斷：

- 0-99：Stage 1。
- 100-279：Stage 2。
- 280-529：Stage 3。
- 530 以上：成熟可採收。

重要：
- 如果一次澆水量足以跨多階段，也不要一次跳過多個階段。
- 例如快從第 1 階跨到第 3 階時，第一次點澆水只進化到第 2 階；下一次點澆水才進入第 3 階。

## 重要 localStorage 狀態

請保護既有資料，不要清空或重置：

- `chromatica.settings.practice`
- `chromatica.settings.sound`
- `chromatica.settings.display`
- `chromatica.dailyLoginBonus`
- `chromatica.dailyWaterState`
- `chromatica.dailyTaskBonus`
- `chromatica.streakMilestoneRewards`
- `chromatica.rainEventState`
- `chromatica.starterPlantSelected`

另外專案內已有花園、目前植物、圖鑑、首頁展示植物、每日目標、練習紀錄等 localStorage key。接手前請先搜尋現有 key，不要另起一套資料結構，也不要覆蓋使用者資料。

## 版本與資源版本參數

- 目前實際版本號請以原始碼內顯示的 `refresh-*` 為準。
- 最近開發過程中曾要求更新到 `refresh-97` 後繼續，但接手時請先用搜尋確認目前實際值。
- 每次完成可見功能或資源變動時，請同步更新：
  - App 版本顯示。
  - `styles.css` 的 query version。
  - `app.js` 的 query version。
  - `service-worker.js` cache name。

建議接手時先搜尋：

```bash
rg "refresh-|styles.css\\?|app.js\\?|CACHE" outputs/chromatica-lab
```
